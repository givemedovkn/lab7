# Задание 1

### Текст задачи
В файле найти среднее арифметическое максимального и минимального элементов.
В текстовом файле хранятся целые числа по одному в строке, исходный файл необходимо заполнить случайными данными, 
заполнение организовать отдельным методом.

### Алгоритм решения
Статический метод заполнения файла случайными данными:
Параметры - количество чисел и путь к файлу. Создается объект, который генерирует случайные числа.

Циклически (выполнится count раз = количеству чисел) генерируется случайное целое число [-1000;1000]. Полученное число записывается в файл в новую строку.

Статический метод поиска среднего арифметического максимального и минимального элементов файла:
Параметр - путь к файлу.
Если файла по заданному пути не существует, вернуть 0.
Создать список из чисел, содержащихся к файле. Если список пустой, то вернуть 0, иначе вернуть сумму минимального и максимального элементов списка, делённую на 2.

# Задание 2

### Текст задачи
Вычислить произведение чётных элементов.
В текстовом файле хранятся целые числа по несколько в строке, исходный файл необходимо заполнить случайными данными, 
заполнение организовать отдельным методом.

### Алгоритм решения
Статический метод заполнения файла случайными данными:
Параметры - количество строк и путь к файлу. Создается объект, который генерирует случайные числа.
Текстовый файл по полученному пути открывается для записи.
Внешний цикл (выполнится count раз = количеству строк): генерируется количество чисел в строке [1, 5]. 
Внутренний цикл(выполнится countNum раз = количеству чисел в строке): генерируется случайное целое число [-20;20]. Полученное число записывается в файл в текущую строку. 
По окончании внутреннего цикла делается переход на новую строку в файле.

Статический метод вычисления произведения четных элементов в файле:
Параметр - путь к файлу.
Если файла по заданному пути не существует, вернуть 0.
Начальное значение произведения = 1. Логический флаг для отслеживания найденных чётных чисел = false.
Считываем все значения файла в массив строк.
Для каждой строки выполняется разделение на части, используя пробел как разделитель и игнорируя лишние пустоты.
Для каждой полученной части проверить, является ли она целым числом.
Если это число, проверить его на чётность и неравенство нулю. 
При соблюдении условий умножить текущее значение произведения на это число и отметить флаг для отслеживания найденных чётных чисел = true.
Если за всё время поиска флаг для отслеживания найденных чётных чисел остался false, вернуть 0, 
иначе итоговое произведение.

# Задание 3

### Текст задачи
Переписать в другой файл строки, в которых нет латинских букв.
В текстовом файле хранится текст.

### Алгоритм решения
Статический метод копирования строк без латинских букв в файл:
Параметры - путь к исходному файлу и путь к файлу, в который будет производиться копирование.
Из исходного файла считываются все строки в массив строк.
Текстовый файл для копирования открывается для записи. 
Обходится каждая строка массива. Определяется логический флаг для отслеживания нахождения 
латинских букв в строке. Обходится каждый символ в строке и проверяется на принадлежность латинскому
алфавиту (A <= символ <= Z). Если нашелся хоть один символ, то флаг = true, выход из цикла.
Если в строке не было найдено латинских символов, то строка записывается в текстовый файл для копирования.

# Задание 4

### Текст задачи
Найти количество квадратов нечетных чисел среди компонент файла.
Бинарные файлы, содержат числовые данные, исходный файл необходимо заполнить случайными данными, 
заполнение организовать отдельным методом.

### Алгоритм решения
Статический метод заполнения файла случайными данными:
Параметры - количество чисел и путь к файлу. Создается объект, который генерирует случайные числа.
Бинарный файл по полученному пути открывается для перезаписи.
Циклически (выполнится count раз = количеству чисел) генерируется случайное целое число [1;100].
Полученное число записывается в файл в новую строку.

Статический метод счета квадратов нечетных чисел:
Параметр - путь к файлу. Если файла по заданному пути не существует, вернуть 0.
Определяется начальное значение количества квадратов = 0.
Бинарный файл открывается для чтения. Пока не дошли до конца файла, 
считываем целое число (4 байта) и проверяем его на квадрат. Если число - квадрат нечетного числа,
значение количества квадратов увеличивается на 1.

Статический метод проверки числа на квадрат нечетного числа:
Параметр - целое число. Если число отрицательное, вернуть false.
Иначе считаем корень от целого числа. Если корень от целого числа в квадрате = целому числу и корень целого числа нечетный,
то вернуть true. В других случаях false.

Статический метод вывода бинарного файла:
Если файла по заданному пути не существует, закончить выполнение метода.
Бинарный файл открывается для чтения. Пока не дошли до конца файла, 
считываем целое число (4 байта) и выводим его на экран.

# Задание 5

### Текст задачи
Файл содержит сведения об игрушках: название игрушки, ее стоимость в рублях и возрастные границы 
(например, игрушка может предназначаться для детей от двух до пяти лет). 
Получить название самой дорогой игрушки для детей до четырех лет.
Бинарные файлы содержат величины типа struct, заполнение исходного файла необходимо организовать отдельным методом, обязательно использовать xml сериализацию, 
решение без сериализации не принимается.

### Алгоритм решения
Структура игрушек вне класса:
Есть 4 поля: название игрушки, стоимость, минимальный и максимальный возраст предназначения.
Есть конструктор с параметрами, принимающий все 4 поля.
Также для кажого поля есть аксесоры.

Статический метод сериализации данных в XML-файл:
Параметры - путь к файлу и список структуры игрушек.
Создаем объект сериализации. Открываем файл для перезаписи. 
Вызвается метод Serialize, который берет список объектов toys 
из оперативной памяти и переводит их в текстовый формат XML.

Статический метод нахождения самой дорогой игрушки для детей до 4х лет:
Параметры - путь к файлу. Создаем объект сериализации.
Объявляем список структуры игрушек. Из файла считываем в список элементы структуры.
Задаем начальное значение для названия самой дорогой игрушки (Не найдено) и максимальную цену (-1.0).
Циклично обходим каждую игрушку в списке, проверяя, чтобы её максимальный возраст предназначения не превышал 4.
Если нашлась игрушка, максимальный возраст которой <= 4 и цена которой больше начального значения
максимальной цены, то эта игрушка - искомое.
После окончания цикла возвращаем название самой дорогой игрушки.
